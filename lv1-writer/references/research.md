# Research — finding what's missing

Never let a draft get built from memory. Given the task and what's already in
`sources/library.md`, find what's missing and write `runs/<id>/01-research.md`:

```
# Research — <task title>

## Key findings
- [A] <fact> — source: <url / citation>
- [B] <fact> — source: <…>
- [C] <unverified note> — needs confirmation

## Conflicts / gaps
<where sources disagree, or evidence is thin>
```

Hard rules: every fact carries a real source you fetched and read — actually search
and fetch, don't write a finding from recollection and grade it A. Grade each A/B/C/D.
Prefer primary sources (papers, official records, original texts) over aggregators.
If sources conflict, present both rather than picking the more convenient one. Never
fabricate a citation.

Once findings are gradable, also append them to `sources/library.md` in the same
format `source-intake` uses, so the draft and inspection stations have one place to
check. Report the most decision-relevant facts back to the user.
