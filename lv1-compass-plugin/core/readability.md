# Readability & format — how a deliverable looks on the page

Read by the **draft** station (which applies it) and the **inspector** (which enforces it).
Single-sourced here so the maker and the judge use the identical rules and can't drift.

The governing principle is **format follows content**: pick each block's shape from what the
content *is*, never from a template or habit. Two failures are equally bad —
**under-formatting** (a scannable set buried in prose) and **over-formatting** (a connected
argument shredded into bullets).

---

## The trigger test (apply to every block)

- **Will the reader scan or refer back to these items individually?** → list or table them.
- **Must they follow a logical thread, each part depending on the last?** → keep prose.

When unsure, name what the reader *does* with the block: looks things up / compares → list
or table; follows an argument → prose.

---

## Use a LIST when…

- **A sentence enumerates 3+ discrete items** → break them out. **Numbered** when order or
  count matters (steps, ranked items, "three reasons"); **bulleted** when it doesn't.
- **A legend or key** — grade definitions, abbreviation keys, status codes — gets **one
  entry per line**, never packed into a paragraph.
- **A long reference / index run-on** (`S01 … · S02 … · S03 …`) → a bulleted list, grouped
  under sub-labels when there are many.
- Each item is **short and parallel** (see Mechanics). The moment an item grows into
  multi-sentence reasoning, it is prose, not a bullet.

## Stack label–value metadata — don't chain it

- **Header fields** (Prepared-for / Date / Role / Author / Version / Status) get **one per
  line**, each as `**Label:** value`. Never chain them with `·`, `|`, or commas.
- A single label–value pair inline in a sentence is fine; **three or more** want stacking.

## Use a TABLE when…

- **Several items share the same dimensions** (each has a cost, a risk, a fit) → one row per
  item, one column per dimension — not repeated parallel bullets.
- **Numbers invite comparison** → align them in a column and put units in the header. A lone
  number stays inline.
- **Multiple if/then branches or decision rules** → a two-column *condition → result* table.
- Don't build a table for a single row or single column — that's just a line of text.

## Use STEPS or ARROWS when…

- **A procedure, sequence, or pipeline** → numbered steps; or arrow notation `A → B → C` for
  a compact flow or cause-and-effect chain. Never narrate an ordered procedure as flat prose.

## Keep PROSE when…

- **Argument, reasoning, narrative, synthesis** — anything the reader follows as a connected
  thread — stays in sentences. Over-listing a flowing argument hurts readability as much as
  under-listing a scannable set. **Reasoning is the one thing you must never bullet.**
- A list of bare terms can be a list; the moment each entry carries a "because", write it as
  prose.

## Other forms

- **Definitions** — bold the term at first use (`**term**`); a *cluster* of definitions → a
  definition list (`term — meaning`, one per line) or a two-column table.
- **A verbatim quote longer than a clause** → a blockquote, not a run-in with quote marks.
- **A caveat, limit, or warning the reader must not miss** → a short standalone callout line,
  not buried mid-paragraph.

---

## Mechanics (apply within whatever form you chose)

- **Parallelism** — items in one list share grammatical form and lead with the load-bearing
  word; put the word the reader scans for first.
- **One kind per list** — a list holds one type of thing. Don't mix a step, a definition, and
  an argument in the same list; switch kind → new list or a table.
- **Item length** — list items stay short and scannable; if they balloon, convert to prose or
  to a table with a notes column.
- **Nesting** — don't nest past two levels; a third level usually means a table or
  sub-sections.
- **Emphasis** — bold the scannable lead-in of a list item or a key term only; never bold a
  whole sentence or use bold as running emphasis.
- **Headings** — chunk anything long, one idea per section. In the **analytical** register a
  heading states the takeaway, not just the topic; in **neutral-professional** it labels the
  section plainly.
- **Paragraphs** — break one that runs past ~5–6 sentences or that covers two distinct ideas.

---

## The balance

Format is a deliberate decision *per block*, not a licence to decorate. The inspector flags
**both** directions: prose hiding a comparison that wanted a table, and bullets shredding an
argument that wanted to be made. Match the form to the content — then stop.
